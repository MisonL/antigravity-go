# Antigravity-Go 阶段五演进蓝图（产品化与生命周期）

## 战略目标
将智能体打造为具备产品化思维的全栈工程师，支持外部数据源、长效任务续传以及自动化部署。

## 1. 外部数据面 [优先级：高]
- [ ] **动态服务器加载**：在 Web 工作台支持动态添加和配置新的 MCP 服务，例如数据库查询与网络搜索。
- [ ] **多源上下文感知**：让智能体具备同时调用多个 MCP 工具解决复杂业务问题的能力，例如查库、分析、改代码和发送通知。

## 2. 任务续传面 [优先级：高]
- [ ] **全状态序列化**：对接 `/GetAllCascadeTrajectories` 与 `/AddTrackedWorkspace`，实现会话级状态快照。
- [ ] **增强热启动**：支持 `ago resume --id [UUID]` 恢复环境变量、未提交差异和思考链路等完整现场。

## 3. 部署与交付面 [优先级：中]
- [ ] **新增 `ago deploy` 子命令**：
  - 自动生成 `Dockerfile` 与持续集成配置
  - 集成云端发布工具，支持一键发布到测试或生产环境
- [ ] **发布前预检**：在部署前强制触发阶段四的 ReviewerAgent 执行上线前审计。

## 4. 全栈脚手架 [优先级：低]
- [ ] **内置模板**：提供 Go + React + SQLite 的标准化全栈模板，支持智能体快速初始化业务项目。

*注：继续坚持 CSE 闭环工程理念，每一项外部集成都必须经过机器预审，确保安全与一致性。*
