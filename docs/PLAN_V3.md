# Antigravity-Go 阶段三扩展蓝图

## 战略目标
将阶段二构建出的六大核心能力面（记忆、轨迹、执行、视觉、工作区、版本）向用户层与业务层释放，打造全方位可视化协同工作台，并探索更高级的自主验证与协作模式。

## 1. 深度可观测性与仪表盘 [优先级：高]
- [ ] **暴露 API 能力**：在 `internal/server/server.go` 中新增 REST API 路由，将 `TrajectoryManager` 和 `MemoryManager` 的能力暴露给前端。
- [ ] **可视化轨迹树**：在 Web 前端接入轨迹接口，以时间轴形式展示智能体的思考链路、工具调用记录与回滚节点。
- [ ] **建立记忆管理面板**：允许用户在 Web 端查看、管理内核自动沉淀的架构决策与项目知识。

## 2. 交互式视觉自测闭环 [优先级：中]
- [ ] **智能体驱动测试**：编写端到端指令，让智能体启动自身的 Web 前端，利用 `browser_open`、`browser_click`、`browser_screenshot` 自主验证界面可用性。

## 3. 人机协同与安全审批 [优先级：中]
- [ ] **前端差异审批**：当智能体准备执行 `apply_core_edit` 时，拦截请求并将 `edit_preview` 发送至 Web 端，等待用户点击“批准”后再真正落盘。

## 4. 多智能体编排 [优先级：低]
- [ ] **生成者与审查者机制**：基于现有 `specialist.go`，实现生成者与审查者对弈。Coder 生成代码，Reviewer 调用 `get_validation_states` 审计；失败则触发 `rollback_to_step`。

*注：继续坚持 CSE 闭环工程理念，每一步扩展都必须具备明确测试支撑。*
